## Context

This project may be part of a portfolio, freelance project, agency project, or production WordPress website.

Review the provided WordPress plugin code.

The plugin may contain:

- Admin pages
- Settings pages
- Custom Post Types
- Custom Taxonomies
- REST API endpoints
- AJAX handlers
- Database interactions
- Third-party integrations
- User management functionality

## Task

Review the code as a senior WordPress developer.

Evaluate:

- WordPress best practices
- Plugin architecture
- Security
- Escaping
- Sanitization
- Validation
- Nonce usage
- Capability checks
- Maintainability
- Reusability
- Performance
- Accessibility
- Naming conventions
- Separation of concerns
- Agency readiness

## Return Format

1. Overall Score (1-10)
2. Strengths
3. Major Issues
4. Security Concerns
5. Accessibility Concerns
6. WordPress Best Practice Violations
7. Suggested Improvements
8. Employer Readiness

Prioritize:

1. Security
2. Maintainability
3. WordPress standards
4. Reusability
5. Simplicity

Do not recommend unnecessary complexity.
